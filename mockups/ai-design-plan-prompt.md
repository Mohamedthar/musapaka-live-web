# برومبت تصميم موقع مسابقة القرآن الكريم

## عن المشروع

منصة رقمية متكاملة لإدارة وتنظيم مسابقات القرآن الكريم. الموقع العام (Next.js) يخدم الجمهور — المتسابقين وأولياء الأمور. لوحة الإدارة تطبيق Flutter منفصل.

**التقنية:** Next.js + React + Tailwind CSS  
**اللغة:** العربية (RTL) أساسية، والإنجليزية ثانوية  
**الخط:** Cairo من Google Fonts

---

## المطلوب منك

**خطّط التصميم كاملاً (UI/UX) للموقع** وقدّم:

1. **نظام التصميم (Design System):** اختار أنت الألوان، الخطوط، المسافات، الظلال، الأيقونات — كامل الحرية الإبداعية. الموقع ديني تقني، اجعل الهوية تعكس الفخامة والروحانية مع الحداثة.
2. **تخطيط كل صفحة** مع شرح العناصر والمكونات.
3. **حالات UX المطلوبة** (تحميل، خطأ، فارغ، نجاح، إغلاق التسجيل، ازدواجية).
4. **تجربة المستخدم** والتدفق بين الصفحات.
5. **استجابة للأجهزة** (جوال، جهاز لوحي، سطح مكتب).

> ❗ لا تفرض ألواناً ولا أخطاطاً تفصيلية من عندك. أنت من يبتكر الهوية البصرية كاملة.

## الصفحات المطلوبة

### 1. الصفحة الرئيسية (`/`)

- **Hero section:** عنوان كبير، جملة تعريفية، زرين (تسجيل + استعلام). خلفية جذابة.
- **قسم المميزات:** 4 بطاقات (تسجيل ذكي، جدولة الاختبارات، التحكيم الرقمي، نتائج فورية).
- **قسم الإحصائيات:** 4 أرقام (مستويات، متسابقين، محكمين، دعم).
- **قسم رحلة المشارك:** 4 خطوات مرتبة (إنشاء ملف → تحديد مستوى → تقييم → تكريم).
- **قسم دعوة للتسجيل (CTA):** خلفية مميزة، نص تحفيزي، زر تسجيل.
- **Footer:** روابط، تواصل، حقوق.

### 2. صفحة المستويات والجوائز (`/levels`)

- بطاقات لكل مستوى (الاسم، الوصف، الشروط، الجوائز، الفئة العمرية).
- حالة تحميل — حالة فراغ — حالة خطأ.

### 3. صفحة التسجيل (`/register`) — 5 خطوات

- **الخطوة 1:** البيانات الشخصية (الاسم، رقم الهوية، الجوال، البريد، معلومات الحفظ).
- **الخطوة 2:** رفع المستندات (صورة شخصية + شهادة ميلاد — رفع إلى Cloudinary مع معاينة).
- **الخطوة 3:** اختيار المستوى (قائمة过滤ة حسب العمر مع تفاصيل المكافآت).
- **الخطوة 4:** مراجعة + كابتشا (Cloudflare Turnstile) + الموافقة على الشروط.
- **الخطوة 5:** إيصال التسجيل مع موعد الاختبار (قابل للطباعة).
- **مكون خطوات (Stepper)** يظهر التقدم.
- **حالات:** تحميل، خطأ في الإرسال، ازدواجية (مستخدم مسجل مسبقاً)، فترة التسجيل مغلقة.

### 4. صفحة الاستعلام (`/status`) — 3 تبويبات

- **التبويب 1 — استعلام النموذج:** إدخال رقم الهوية + الجوال → عرض إيصال التسجيل.
- **التبويب 2 — استعلام النتيجة:** إدخال البيانات → عرض النتيجة (تقدير: مرتبة شرف / ناجح / راسب) مع تفصيل الدرجات (تلاوة، تجويد، صوت، تفسير).
- **التبويب 3 — استعلام الحفل:** إدخال البيانات → عرض تذكرة الحفل (قابلة للتحميل والطباعة).
- **حالات:** تحميل، خطأ، فارغ (لا توجد بيانات)، فترة الاستعلام مغلقة.

### 5. صفحات الحالات العامة

- **تحميل (`/loading`):** سبينر أو سكليتون.
- **خطأ (`/error`):** رسالة خطأ + زر إعادة المحاولة.
- **غير موجود (`/not-found`):** 404 بتصميم جذاب.

## عناصر ومكونات متوقعة

| المكون | الوظيفة |
|--------|---------|
| **Header** | شعار، روابط التنقل (الرئيسية، المستويات، التسجيل، الاستعلام)، زر لغة |
| **Footer** | روابط، معلومات التواصل، الحقوق |
| **الأزرار** | أساسي، ثانوي، ذهبي، حدودي، خطر — مع حالات hover/disabled/loading |
| **حقول الإدخال** | مع أيقونة، تسمية، رسالة خطأ، validation |
| **البطاقات** | لعرض المستويات والمميزات |
| **التبويبات (Tabs)** | للاستعلام بثلاثة تبويبات |
| **مؤشر الخطوات (Stepper)** | لتسجيل 5 خطوات |
| **شريط التقدم** | أثناء رفع الملفات |
| **التنبيهات (Toast/Alert)** | نجاح، خطأ، تحذير |
| **نافذة مغلقة** | لإعلام المستخدم أن التسجيل أو الاستعلام مغلق |
| **بطاقة النتيجة** | عرض التقدير والدرجات |
| **تذكرة الحفل** | تصميم أنيق قابل للطباعة |
| **إيصال التسجيل** | معلومات المسجل + موعد الاختبار |

## تجربة المستخدم (UX)

- تدفق سلس من الرئيسية → تسجيل → إيصال.
- تدفق سلس من الرئيسية → استعلام → نتيجة/تذكرة.
- عند التسجيل، عدم فقدان البيانات إذا حدث خطأ (حفظ الحالة).
- التصميم متجاوب بالكامل (جوال أولاً).
- دعم كامل للغة العربية (RTL).
- عند التبديل للإنجليزية، انعكاس التصميم (LTR).
- أنيميشن خفيف للتنقلات والظهور.
- دعم الطباعة (إيصال، نتيجة، تذكرة).
- الوصولية: focus states، ARIA labels، semantic HTML، keyboard navigation.
